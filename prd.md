# 4지선다 객관식 퀴즈 게임 - 프로덕트 요구사항 명세서 (PRD)

## 1. 개요

### 1.1 게임 개요
한국사, 과학, 지리, 일반상식 4개 카테고리의 4지선다 객관식 퀴즈 게임입니다.
사용자가 40개의 문제를 풀면서 지식을 점검하고, 다른 사용자와의 순위 경쟁을 통해
**학습**과 **오락**, **경쟁** 요소를 동시에 즐길 수 있습니다.

### 1.2 핵심 특징
- **한판 플레이**: 10~20분 내에 완결되는 빠른 게임
- **모바일 최적화**: 웹과 모바일 모두 완벽 지원
- **온라인 기반**: 실시간 순위 기록 및 동기화
- **개발자 대상**: 소프트웨어 전공자, 개발자를 위한 기술 면접 및 코테 대비 게임

### 1.3 타겟 사용자
성인 일반 (직장인, 대학생 등) - 지식 점검과 일시적 오락을 원하는 사용자

### 1.4 게임 가치 제안
- 짧은 시간에 지식을 점검할 수 있음
- 다른 사용자와 경쟁할 수 있음 (순위 시스템)
- 캐주얼하면서도 의미 있는 콘텐츠 제공

## 2. 게임 규칙

### 2.1 기본 규칙
- **형식**: 4지선다 객관식 퀴즈
- **카테고리**: 자료구조, 알고리즘, 코딩테스트, 시스템/네트워크 (4개)
- **총 문제 수**: 40문제
- **카테고리별 문제 수**: 10문제씩
- **난이도**: 카테고리별 중상 수준 (혼합)

### 2.2 게임 진행 방식
1. 게임 시작 후 첫 번째 카테고리(한국사) 부터 순차 진행
2. **문제당 30초 시간 제한** - 시간 초과 시 자동으로 다음 문제로 진행
3. 사용자가 4개 선택지 중 1개 선택
4. **선택 직후 즉시 정답/오답 피드백 + 정답 공개**
5. 모든 카테고리 완료 후 최종 점수 계산

### 2.3 사용자 조작
- **문제 건너뛰기**: 시간 초과 전 "건너뛰기" 버튼으로 부답 처리 후 다음 문제로 진행 가능
- **돌아가기**: 카테고리 내에서만 이전 문제로 다시 풀기 가능 (재답변 가능)
- **다시 풀기**: 모든 40문제 완료 후 게임 재시작 가능

### 2.4 점수 시스템
- **정답**: 1점 부여
- **오답 또는 미답변**: 0점
- **최고 점수**: 40점
- **정답률**: (정답수 / 40) × 100%

### 2.5 피드백
- **즉시 피드백**: 각 문제 선택 직후 정답/오답 여부 표시
- **정답 공개**: 오답/건너뛰기한 경우 정답 표시
- **설명**: 각 정답에 대한 간단한 설명 제공 (선택)

## 3. 주요 기능

### 3.1 게임 화면
- **현재 진행 상황 표시**
  - 현재 문제 번호 (예: 12/40)
  - 현재 카테고리 표시
  - 진행률 바 (비주얼)
- **타이머**: 각 문제당 남은 시간 (30초)
- **문제와 선택지**: 4개 선택지 클릭 가능
- **조작 버튼**: 건너뛰기, 돌아가기

### 3.2 결과 화면
- **최종 점수**: 큰 폰트로 강조 (예: 32/40)
- **정답률**: 백분율로 표시 (예: 80%)
- **카테고리별 성적**: 각 카테고리의 정답 수
  - 한국사: 8/10
  - 과학: 9/10
  - 지리: 7/10
  - 일반상식: 8/10

### 3.3 순위 기록 시스템
- **로그인 옵션**: 계정으로 로그인하여 순위 자동 기록
- **게스트 옵션**: 닉네임만 입력하여 순위 기록
- 점수와 함께 자동 저장
- 역대 순위 조회 기능 (상위 50명)
- 나의 순위 확인 가능

## 4. 카테고리 상세

### 4.1 카테고리 구성

| 카테고리 | 문제 수 | 난이도 | 주요 내용 |
|---------|--------|--------|---------|
| 자료구조 | 10 | 중 | Array, LinkedList, Tree, Hash, Graph 등 |
| 알고리즘 | 10 | 중 | 정렬, 탐색, DP, 그리디, 분할정복 등 |
| 코딩테스트 | 10 | 중상 | 코테 문제 풀이, 시간복잡도, 최적화 등 |
| 시스템/네트워크 | 10 | 중 | OS, 네트워크, DB, 보안 등 |

### 4.2 세부 구성

**자료구조 (10문제)**
- Array & List: 2-3문제
- Tree & Binary Tree: 2-3문제
- Hash & Set: 2문제
- Graph: 2-3문제

**알고리즘 (10문제)**
- 정렬: 2문제 (QuickSort, MergeSort 등)
- 탐색: 2문제 (BFS, DFS, Binary Search 등)
- 동적 프로그래밍: 2-3문제
- 그리디/분할정복: 2-3문제

**코딩테스트 (10문제)**
- 문제 해석: 2문제
- 시간/공간 복잡도: 2-3문제
- 엣지케이스 처리: 2-3문제
- 최적화 기법: 2-3문제

**시스템/네트워크 (10문제)**
- OS (프로세스, 메모리, 스레드): 3문제
- 네트워크 (TCP/IP, HTTP): 3문제
- 데이터베이스 (SQL, 인덱싱): 2문제
- 보안 (암호화, 해싱): 2문제

## 5. 기술 요구사항

### 5.1 기술 스택

**프론트엔드**
- **Framework**: React 18+
- **상태 관리**: Redux 또는 Zustand
- **스타일링**: Tailwind CSS 또는 styled-components
- **UI 컴포넌트**: Material-UI, Ant Design 등 (선택)
- **빌드 도구**: Vite 또는 Create React App

**백엔드**
- **Runtime**: Node.js 18+
- **Framework**: Express.js
- **API**: RESTful API 또는 GraphQL (선택)
- **인증**: Firebase Authentication

**데이터베이스 & 호스팅**
- **Database**: Firebase Firestore / Realtime Database
- **파일 저장**: Firebase Storage
- **호스팅**: Firebase Hosting
- **인증**: Firebase Authentication

### 5.2 성능 요구사항
- **페이지 로딩**: 3초 이내
- **API 응답**: 1초 이내
- **점수 동기화**: 실시간 (2초 이내)
- **모바일 최적화**: Core Web Vitals 기준 만족

### 5.3 브라우저 지원
- Chrome, Safari, Firefox, Edge 최신 버전
- Mobile Safari (iOS 12+)
- Chrome Mobile (Android 5+)

### 5.4 보안 요구사항
- **HTTPS**: 모든 통신 암호화
- **인증**: Firebase Authentication
- **데이터 보호**: Firestore Security Rules 설정
- **CORS**: 필요시 설정

### 5.5 데이터 저장
- **사용자 성적**: Firestore에 저장
  - userId, nickname, score, answers, timestamp 등
- **순위 데이터**: Firestore에서 실시간 조회
- **로그인 사용자**: Firebase Authentication
- **게스트 사용자**: 임시 세션 (로컬 스토리지 병행)

### 5.6 사용자 경험

## 6. 추가 기능

### 6.1 우선순위 높은 기능 (Phase 1)

#### 1. 난이도 선택
- **기능**: 게임 시작 전 난이도 선택 가능
- **구성**: 쉬움 / 보통 / 어려움 (각각 40문제씩)
- **설명**: 사용자가 원하는 난이도의 40문제를 선택해서 플레이
- **점수**: 난이도별로 순위 분리 관리

#### 2. 리플레이/복습 모드
- **기능**: 게임 완료 후 학습 모드로 진입
- **옵션 1**: 틀린 문제만 다시 풀기
- **옵션 2**: 특정 카테고리의 모든 문제 복습
- **점수 반영**: 복습 모드는 순위에 영향 없음

#### 3. 통계/분석
- **기본 통계**: 총 점수, 정답률, 카테고리별 성적
- **상세 분석**:
  - 강점/약점 카테고리 표시
  - 성적 추이 그래프 (최근 10게임)
  - 평균 정답 시간, 오답률
- **역사 데이터**: 게임 기록 조회 (날짜, 점수, 난이도별)

#### 4. 힌트 기능
- **기능**: 각 문제당 힌트 사용 가능
- **구성**:
  - 선택지 중 명백히 틀린 선택지 제거 (1개)
  - 무제한 사용 가능
- **점수 감점**: 힌트 사용 시 0.5점 감점
- **예시**: 힌트 사용해서 정답하면 0.5점만 획득

## 7. 대표 캐릭터: 띠띠

### 7.1 캐릭터 개요
- **이름**: 띠띠 (Tti-Tti)
- **종류**: 귀여운 수달
- **성격**: 친근하고 응원해주는 학습 가이드
- **역할**: 게임 진행 가이드, 격려/응원, 반응형 대사

### 7.2 캐릭터 특징
- **외형**: 둥글고 귀여운 수달 모양
- **색상**: 따뜻한 갈색 또는 브라운톤
- **표정**: 다양한 감정 표현 (행복, 응원, 위로, 생각 중 등)

### 7.3 캐릭터 역할

**게임 시작 전**
- "안녕! 나는 띠띠야. 함께 퀴즈를 풀어볼까?"
- 난이도 선택 시 격려: "와! 넌 도전적이네! 화이팅! 💪"

**게임 진행 중**
- 정답 시: "오오! 정답이야! 잘했어! 🎉"
- 오답 시: "아쉬워! 하지만 잘했어. 다음 문제 파이팅! 💪"
- 건너뛰기: "괜찮아, 다음 문제 가자!"
- 시간 초과: "시간이 다 됐어. 다음 문제로 가볼까?"

**게임 완료 후**
- "완료했어! 너 정말 잘했어! 🌟"
- 고득점 시: "와아! 이건 뛰어난 점수야! 천재네! 🏆"
- 저득점 시: "괜찮아, 계속 연습하면 더 잘할 수 있어! 💪"

### 7.4 UI 배치
- **게임 화면**: 우측 하단 또는 좌측 상단에 배치
- **결과 화면**: 중앙 상단에 큰 캐릭터 + 응원 메시지
- **메뉴**: 왼쪽 사이드 또는 하단 네비게이션
