## 📊 추가된 새로운 문제 (5개)

### 문제 41: AVL Tree의 특징 (자료구조, Hard)
```
문제: AVL Tree의 특징은?
선택지:
  1. ✅ 각 노드의 왼쪽, 오른쪽 서브트리의 높이 차이가 최대 1
  2. 모든 노드의 높이가 정확히 같은 트리
  3. 완전 이진 트리
  4. 완전 이진 탐색 트리
해설: AVL Tree는 자가 균형 이진 탐색 트리로, Balance Factor가 -1, 0, 1 중 하나여야 함
```

### 문제 42: 동적 프로그래밍의 필수 조건 (알고리즘, Hard)
```
문제: 동적 프로그래밍(DP)의 필수 조건이 아닌 것은?
선택지:
  1. 최적 부분 구조(Optimal Substructure)
  2. 중복되는 부분 문제(Overlapping Subproblems)
  3. ✅ 그래프의 순환 구조
  4. 메모이제이션 또는 태뷸레이션
해설: DP는 최적 부분 구조와 중복 부분 문제를 가져야 하며, 순환 구조는 필수 아님
```

### 문제 43: 코드의 시간복잡도 (코딩테스트, Hard)
```
문제: 다음 코드의 시간복잡도는?
선택지:
  1. O(n)
  2. O(n log n)
  3. ✅ O(n²)
  4. O(2ⁿ)
해설: 중첩 반복문이 있을 때 외부 루프 n번, 내부 루프 n번 실행되므로 O(n²)
```

### 문제 44: TCP와 UDP의 차이 (시스템/네트워크, Hard)
```
문제: TCP와 UDP의 차이 중 TCP의 특징이 아닌 것은?
선택지:
  1. 연결 기반(Connection-oriented)
  2. 신뢰성 보장
  3. ✅ 브로드캐스트 지원
  4. 순서 보장
해설: TCP는 연결 기반, 신뢰성, 순서 보장 제공. UDP는 브로드캐스트 지원
```

### 문제 45: SQL의 JOIN (시스템/네트워크, Hard)
```
문제: SQL의 JOIN 종류 중 양쪽 테이블에 모두 있는 데이터만 반환하는 것은?
선택지:
  1. LEFT JOIN
  2. RIGHT JOIN
  3. ✅ INNER JOIN
  4. FULL OUTER JOIN
해설: INNER JOIN은 양쪽 교집합만 반환. LEFT/RIGHT는 한쪽 모두, FULL은 양쪽 모두
```

## 🛠️ 명령어 사용 방법

### 1. 새 문제 추가 (2단계 프롬프트)
```bash
npm run quiz-add
```
**프로세스:**
1. 카테고리 선택 (1-4)
2. 난이도 선택 (easy/normal/hard)
3. 문제 입력
4. 문제 설명 입력
5. 선택지 4개 입력
6. 정답 선택 (1-4)
7. 정답 해설 입력
8. 최종 확인 후 저장

### 2. 문제 검증
```bash
npm run quiz-validate
```
- 모든 문제의 형식, 콘텐츠, 난이도 검증
- 검증 결과 요약 및 통계 출력
- 실패한 문제 목록 및 오류 상세 표시

### 3. 난이도 분포 분석
```bash
npm run quiz-range
```
- 전체 문제 통계
- 카테고리별 분포
- 난이도별 분포
- 밸런스 평가
- 새로 추가된 문제 현황
