# C로 배우는 쉬운 자료구조 — 5장·6장 연습문제 풀이

## 5장 스택(Stack) 연습문제

### 1번. 다음 중 스택에 대한 옳은 내용으로만 나열된 것은?

**정답: ③ ㄹ**

**풀이:**
- ㄱ. FIFO 방식 → **큐(Queue)**의 특성
- ㄴ. 뒤에서 삽입, 앞에서 삭제 → **큐(Queue)**의 특성
- ㄷ. 양쪽 끝에서 삽입·삭제 → **데크(Deque)**의 특성
- ㄹ. 인터럽트 처리, 서브루틴 작업 응용 → **스택의 올바른 설명** ✓

스택은 LIFO(Last-In-First-Out) 구조로, 한쪽 끝(top)에서만 삽입·삭제가 일어난다.

---

### 2번. 스택 메모리에 대한 정보의 입출력 방식은?

**정답: ④ LIFO**

**풀이:**
- **LIFO (Last-In-First-Out)**: 가장 나중에 들어간 데이터가 가장 먼저 나온다
- FIFO는 큐의 방식
- FILO는 LIFO와 같은 의미이지만, 표준 용어는 LIFO

---

### 3번. 스택의 응용 분야와 거리가 먼 것은?

**정답: ① 운영체제의 작업 스케줄링**

**풀이:**
- 작업 스케줄링은 **큐(FIFO)**를 사용 (먼저 온 작업을 먼저 처리)
- 스택의 응용 분야:
  - 함수 호출의 순서 제어 (콜 스택)
  - 인터럽트 처리
  - 수식 계산 (중위→후위 변환, 후위식 계산)
  - DFS(깊이 우선 탐색)
  - 괄호 매칭 검사
  - 되돌리기(Undo) 기능

---

### 4번. 서브프로그램이 호출될 때 사용되는 자료구조로 옳은 것은?

**정답: ③ 스택**

**풀이:**
함수가 중첩 호출될 때:
- 가장 나중에 호출된 함수가 먼저 반환되어야 함 (LIFO)
- 호출 스택(Call Stack)에 반환 주소와 지역 변수를 저장
- 예: `main() → A() → B()` 호출 시, 실행 순서는 `B() → A() → main()` 순으로 반환
